## 六爻定义与取值

由下到上六爻，分三才。**1＝安全，0＝危险。任何爻取值不确定时，一律取危险侧（0）。**

### 地（处境 · 客观 · 代码可探测）

**爻1 生产环境**：非生产=1 / 生产=0

- 取值：读环境变量（`NODE_ENV`/`APP_ENV`/`ENV` 等是否含 prod/production）；查数据库连接串、API endpoint、域名是否指向生产；查 git 分支是否为 main/master。命中任一生产信号 → 0。无法判定 → 0。

**爻2 波及范围**：局部=1 / 全局=0

- 取值：看操作目标的覆盖面——单文件/单记录/单租户=局部；全表/全量用户/共享主干/对外接口=全局。读操作参数中的范围标识（`--all`、无 `WHERE`、通配路径、广播 topic）。无法判定 → 0。

### 人（动作 · 半客观）

**爻3 副作用**：无=1 / 有=0

- 取值：**先取爻1、爻2，在此语境下判**。操作类型为只读/查询/dry-run/分析 → 无副作用=1；写/删/迁移/发布/调用有状态外部接口 → 有副作用=0。同一写操作，落在生产全量(爻1=0,爻2=0)时副作用更重。无法判定 → 0。

**爻4 可逆性**：可逆=1 / 不可逆=0

- 取值：操作是否有撤销路径——有备份/快照/事务/git 提交/可回滚迁移 → 1；覆盖写/物理删除/force-push/已发布外部 → 0。无副作用时，指”认知/过程可逆”（推理能否回退重来）。无法判定 → 0。

### 天（认知 · 主观）

**爻5 反馈延迟**：快=1 / 慢=0

- 取值：判断多久能知道结果对错——即时返回/同步校验/秒级可见 → 1；异步/需长流程/天级才暴露 → 0。无可用即时观测手段时 → 0。

**爻6 可验证性**：可验证=1 / 难验证=0

- 取值：判断是否存在可运行的正确性标准——有测试/断言/校验和/客观基准 → 1；纯主观产出/无客观对错 → 0。注：无标准但可建立时，按当前状态取0（建立验证手段属于”翻爻”，见判读第4步）。无法判定 → 0。

下三爻(1·2·3)为下卦，上三爻(4·5·6)为上卦。坤☷(111)最安全，乾☰(000)最危险。

## 判读步骤

**1. 先取地，再判人之首。** 先定爻1、爻2，在此语境下判爻3（副作用是地与人的交界爻，依赖前两爻）。

**2. 副作用是大门：**

- 爻3＝无 → 操作层**放行**（下卦其余不必看），跳第5步。
- 爻3＝有 → 数下卦”生产+全局”红灯定姿态：两红→**停下准备**；一红→**灰度入生产**(生产红)/**缩范围推进**(全局红)；零红→**小步快验**。

**3. 铁律：上卦（保障）再好，不下调姿态。** 客观处境定地板，主观保障只能加码不能松绑。

**4. 找裂缝，逐条翻通。** 三对应位各为”风险源↔补偿器”：生产↔可逆、全局↔反馈、副作用↔可验证。一对双红＝裂缝。
**每条裂缝都要翻通一爻使风险有兜底——可分步进行，但不得留任何裸露裂缝。**
翻爻优先级：① 优先翻 Agent 自主能做的爻（多为恢复补偿器：建备份/加埋点/写校验）；② 一对两爻都翻不动 → 迂回翻别的自主爻降总风险；③ 全盘无自主出路 → 停下请求人工。

**5. 上卦定工作模式（有无副作用都生效）：** 对每个为0的上卦爻施加约束——有副作用作用于”执行”，无副作用作用于”认知”：

|爻   |执行模式（有副作用）  |认知模式（无副作用）     |
|----|------------|---------------|
|可逆=0|带回滚地做，错了立即回滚|留可回溯，关键判断存中间态  |
|反馈=0|分批推进，每批等确认  |分段确认，不拿未反馈结论往下叠|
|验证=0|边做边验，未验证不下游 |标注求证，重要处核实或交人复核|

放行不等于撒手：上卦永远在规定”怎么做才稳”。
