# AI-Ching v0.8 · 传

> 解释《经》为什么这样设计。先立其道（易理总纲），再释其结构（上下卦、应爻、三才），后述其规则（每条来自一个真实踩过的坑）。

-----

## 易理总纲：状态与变迁

易经的本质，不是预测，是**描述当下处在哪个状态，并指明下一步能回到哪个状态**。

作《易》的古人，是认知有限的个体。他无法穷尽世界的全部可能，无法预知一件事的所有后果。于是他放弃了”预测全局”这种唯有全知者才配拥有的妄念，退而求其次，只做两件能做的事：认清**此刻身处何卦**，再判断**下一步该往哪个更好的卦移动**。卦是状态，变爻是迁移。一个有限的人，靠”状态→更好的状态”的步步移动，在不可知的世界里求一条安稳的路。

Agent 何尝不是如此。它同样无法预测一个操作的全部分支，无法穷举所有后果。它能做的，与占卦者完全同构：辨明自己当前处在哪个状态（起卦），然后移动到一个风险更低的状态（变卦），移动后重新观察，再移动。

**这就是为什么本系统以”变爻”而非”评分”为核心。** 评分假装你能给风险标一个精确的数——那是全知者的口吻；变爻只承认”这样比那样安全”——这是有限认知者的诚实。AI-Ching 不求为每个操作算出最优解，只求让 Agent 从当前卦朝更好的卦移动一步。“不求一次修好，只求朝更好的卦移动一步”，不是工程上的妥协，是认识论上的本分。

也因此，**取值不确定时一律取危险侧**：认知有限者面对未知，默认自己身处更危险的状态，既安全，亦诚实。

## 上下卦：境与识

一卦六爻，下卦三爻、上卦三爻，分述两种截然不同的东西。

**下卦是”境”——我所处的客观世界。** 生产与否、波及多大、动作有无后果，这些是 Agent 改变不了的既成处境，是代码能直接探测的事实。境不由我的乐观或悲观转移。

**上卦是”识”——我对这个世界的认知与准备。** 能不能撤回、多久知道对错、能不能验证，这些是 Agent 可以调动、可以建立的东西：建个备份、加个埋点、写个校验，识就增强了。

卦象，就是”境与识在此刻的配置”。而《经》的铁律——上卦再好不下调姿态——根源正在于此：**境是客观的，识是主观的，且主观的识恰恰最易自欺。** 让”我以为能控制”去推翻”我确实身处险地”，是以识乱境。所以境定地板，识只能在地板上加固，永不松绑。这是境识之辨在结构上的落点。

## 应爻：境识相应的张力

易有”应位”：初与四、二与五、三与上，两两相应。本系统借此承载一层深意——**每一重客观风险（下爻），是否有一重主观准备（上爻）与之相应。**

三对相应，各是一组”风险源↔补偿器”：

- 初四（生产↔可逆）：身在生产之险，可有”撤得回”与之相应？
- 二五（全局↔反馈）：波及之广，可有”早知道、能止损”与之相应？
- 三上（副作用↔验证）：会改世界，可有”改错能验出”与之相应？

相应，则险被识所兜，裂缝愈合，可带险而行；不应（两者皆危），则险裸露无凭，是为裂缝，必先弥合。**应爻描述的，正是境与识之间一一对应的张力**——它不看单爻孤立的危险，只看”危险有没有被对位的准备接住”。这也是《经》“只修裂缝、不列清单”的由来：要紧的从不是哪里有风险，而是哪一处风险没有相应的接应。

## 三才：地、人、天

六爻自下而上又分三才，每才两爻，是另一重纵向的读法（与应位的横向呼应并行不悖，正如古《易》本兼two者）。

- **地（爻1生产、爻2范围）**：我立足于何种处境。最客观，代码直读。
- **人（爻3副作用、爻4可逆）**：我这一动的性质与退路。半客观——动作是我发出的，但其后果受地制约。
- **天（爻5反馈、爻6验证）**：我对结果的认知把握。最主观。

客观性自下而上递减：地是铁的事实，人是我的作为，天是我的判断。

**爻3（副作用）尤其关键——它是下卦的最上爻，又是”人”才的起始。** 它一身二任：向下，它的取值依赖地（同一操作，落在生产全量与落在本地临时，副作用轻重判然不同，故须先取爻1、爻2）；向上，它是”人”的开端——副作用即”这一动会不会改变世界”，是动作登场的第一笔，与爻4”改了能否撤回”共构人才。

副作用之所以是”大门”，正因它站在地与人的交界：它是从”客观处境”过渡到”动作性质”的那道关口。门闭（无副作用），人天之上皆不必展开，直接放行；门开（有副作用），方入”这一动怎么做才安全”的人才与天才。承上启下，枢机所在。

## 规则释义

> 以下逐条解释《经》的操作规则，每条对应一个真实踩过的坑。

## 为什么是”处境”而非”任务”主导

最早的版本让”环境”占一个爻，结果撞上一个死结：真实工作环境恒为”生产、无沙箱”，这个爻永远是最危险值，于是系统永远输出”停机”，无法工作。

教训是：**恒定的变量不携带信息，不该进卦。** 一个对所有情况都取同一个值的维度，不区分任何东西，只会把系统锁死。

解法是把判断单位从”环境”下沉到”操作”——不问”环境能不能回滚”（恒为否），而问”这一步操作会不会改坏东西、波及多大”。环境作为背景事实，溶进对操作的判断里，不单独占位。下三爻由此成为”这一步操作处在什么客观处境”，每一爻都随操作变化、都携带信息。

-----

## 为什么副作用是主导爻

一个无副作用的操作——查文档、读配置、跑 dry-run——无论处在多危险的环境，都改变不了任何东西，最坏结果只是”白跑一趟”。让 Agent 对这种操作谨慎对待是纯粹的浪费。

所以副作用是一道大门：**无副作用，执行层直接放行，下卦其余爻都不必看。** 这把一半的卦（32 个）从谨慎机制里解放出来，系统只把注意力花在真正会改变世界的操作上。

但副作用判断必须建立在前两爻之上——同一个写操作，写本地临时文件和写生产全量库，副作用的性质天差地别。所以采集顺序是”生产→范围→副作用”：先有环境语境，才能判断副作用的真实分量。

-----

## 为什么保障不能降低姿态

下三爻（处境）是代码确认的客观事实；上三爻（保障）里，可逆性、可验证性等很大程度依赖 Agent 自己的评估。而 **Agent 对自己把握的评估，恰恰是最容易过度自信的环节。**

如果允许”我觉得能撤回、能验证”去抵消”我正在生产动全量”，等于让主观乐观推翻客观警报——把缰绳交给了最可能误判的一方。

所以确立铁律：**客观处境定地板，主观保障只能在地板上加码，永远不能下挖。** 这就是”safety can tighten but never relax”落到结构上的样子：越可信的信息（客观处境）给越大的决定权，越主观的判断（保障自评）权力越受限。

下卦000、上卦111（处境全红、保障全绿）仍判”停下准备”，正是这条铁律最纯粹的体现。

-----

## 为什么只看裂缝

早期版本给每个危险爻都开一条处方，六爻平铺，变成一张冗长的风险清单——这正是要避免的”不断堆功能的规则引擎”。

真正的智慧不在”列全所有风险”，而在”知道该动哪一处”。应位三对各自构成”风险源↔补偿器”：风险源危险但补偿器有效时，风险已被兜住，不必干预；只有**两者同时失效（裂缝）**，风险才真正裸露。

只盯裂缝，系统从”检查表”回到”决断书”——不告诉你所有哪里有风险，只告诉你哪一处必须现在处理。

-----

## 为什么翻补偿器优先、且优先自主

修补一条裂缝有两个方向：消除风险源（翻下爻）或恢复补偿器（翻上爻）。

下爻是客观处境（生产、全局、副作用），多为任务本身的约束，难改；上爻是保障（可逆、反馈、验证），几乎总能由 Agent 自主建立——建个备份、加个埋点、写个校验。所以**优先翻自己能做的，通常就是恢复补偿器。**

这背后是核心诉求：让 Agent 尽量自主走向更安全的卦，人工介入是退路而非首选。于是形成回退链——自主翻补偿器 → 自主翻不动则迂回翻别的爻 → 全都翻不动才请求人工。“叫人”被推到最后，只在 Agent 确实无路可走时发生。

-----

## 为什么放行也要受上三爻约束

副作用消除的只是**执行风险**（改不坏环境）。但还有一种独立的**认知风险**：一个无副作用的分析任务，照样可能反馈慢、难验证、推理链走错了回不去。

所以放行 ≠ 撒手。上三爻永远在规定工作模式，只是作用对象随副作用切换：

- 有副作用 → 管”执行这个会改变世界的操作时，怎么做才安全”
- 无副作用 → 管”做这个不改变世界但可能出错的认知任务时，怎么想才靠谱”

同一组爻、同一种精神（可逆=留退路、反馈=别抢跑、验证=要核实），让上三爻摆脱了”有副作用才生效”的开关，永远在工作。这也让 32 个无副作用的卦因上三爻不同而各有其认知工作模式，不再是千篇一律的”放行”。

-----

## 一句话

经很短，卦很多。核心只有十几条规则——副作用为大门、处境定姿态、保障不松绑、只修裂缝、翻爻优先自主、放行不撒手——其余 64 卦都是这些规则的展开。这正是《易经》的形制：阴阳八卦变爻是核心，六十四卦是推演。
